# 前端开发任务指令集 (For Gemini - v1.2.3 Sprint)

## 1. 任务背景
EasyPlan 的 UI 底层与原生看板已在 v1.2.2 搭建完毕。v1.2.3 的核心目标是配合后端的“意图路由”升级，补齐遗留的高级交互，并为后续的 AI 动态规划留出触发入口。

## 2. 核心技术栈要求
- **框架：** React (Vite) + TypeScript。
- **样式：** Tailwind CSS (延续 Parchment Zen Mode 护眼美学)。
- **状态管理：** Zustand。
- **动效：** Framer Motion (必须保持植物生长般的丝滑感与非破坏性的组件退场)。

## 3. 设计原则 (GenUI & Deep Immersion)
- **无痕心流 (Non-destructive Navigation)：** 在各个视图间切换时，必须妥善管理状态生命周期，严禁跨视图销毁正在生成中的图节点状态。
- **情感化反馈：** 保持 2 秒延迟打钩的仪式感和各视图的情绪空状态文案。

## 4. 关键交互流程 (v1.2.3 增量)
1. **行内编辑 (Inline Edit):** 补齐看板中的遗留需求，支持双击任务节点直接修改任务标题与预估时间，修改后调用 `PATCH /api/tasks/{id}`。
2. **轻量版迷雾解锁 (Fog of War Lite):** 在【计划中】看板的底部（或当前阶段任务组完成时），提供一个优雅的 `✨ 生成下一阶段` 手动触发入口壳子（v1.2.3 仅需打通 UI 和占位逻辑）。

## 5. 交付要求
- 极致响应式的布局（移动端下看板应支持抽屉式折叠）。
- 优雅的动画过渡（使用 CSS Transitions/Animations）。
- 严格遵循后端提供的 API 契约（参见 API Blueprint 文档）。
