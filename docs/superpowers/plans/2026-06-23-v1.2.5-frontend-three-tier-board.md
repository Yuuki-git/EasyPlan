# EasyPlan v1.2.5 Frontend Three-Tier Board Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Present Roadmap, Current Phase, and Next Action inside the existing EasyPlan TaskBoard and support safe same-thread phase preview, confirmation, cancellation, and recovery.

**Architecture:** Keep the current sidebar/header/task-card system, add a focused `PlanningOverview` component, and isolate all phase selection logic in pure helpers. The Zustand store owns API/SSE lifecycle and persisted recovery keys; components render derived state and never infer completion from unrelated or manual tasks.

**Tech Stack:** React 18, TypeScript, Zustand, Tailwind CSS, Framer Motion, Lucide React, Vite, Node assert tests

- [ ] **Step 3: Implement one derived view**

Export this stable interface and selector:

```typescript
export interface PlanningView {
  context: PlanningContext;
  currentTasks: TaskResponse[];
  historicalPhases: Array<{ phase: RoadmapPhase; tasks: TaskResponse[] }>;
  nextAction: TaskResponse | null;
  totalAiActions: number;
  completedAiActions: number;
  canUnlock: boolean;
  isGoalComplete: boolean;
}

export function selectPlanningView(
  taskTree: TaskTree | null,
  tasks: TaskResponse[],
  threadId: string | null,
): PlanningView | null {
  const context = taskTree?.planning_context;
  if (!context || !threadId) return null;
  const threadTasks = tasks.filter((task) => task.thread_id === threadId);
  const phaseId = context.current_phase?.phase_id ?? null;
  const currentTasks = threadTasks.filter(
    (task) => (phaseId !== null && task.phase_id === phaseId) || task.source === 'manual',
  );
  const aiActions = currentTasks.filter(
    (task) => task.source === 'ai' && task.phase_id === phaseId && task.node_type === 'action',
  );
  const completedAiActions = aiActions.filter((task) => task.status === 'completed').length;
  const nextAction = threadTasks.find(
    (task) => task.client_node_id === context.next_action_client_node_id,
  ) ?? null;
  const completedPhases = context.roadmap.filter((phase) => phase.status === 'completed');
  return {
    context,
    currentTasks,
    historicalPhases: completedPhases.map((phase) => ({
      phase,
      tasks: threadTasks.filter((task) => task.phase_id === phase.phase_id),
    })),
    nextAction,
    totalAiActions: aiActions.length,
    completedAiActions,
    canUnlock: context.current_phase !== null && aiActions.length > 0 && completedAiActions === aiActions.length,
    isGoalComplete: context.current_phase === null && context.roadmap.every((phase) => phase.status === 'completed'),
  };
}
```

Sort phase/task arrays by `order/phase_order`, `sort_order`, and id. Export a separate `buildTaskTree(tasks)` helper so `TaskBoard` uses the same deterministic tree reconstruction for current and historical task sets.
