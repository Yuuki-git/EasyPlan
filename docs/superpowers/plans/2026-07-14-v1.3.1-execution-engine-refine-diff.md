# EasyPlan v1.3.1 Execution Engine / Refine Diff Implementation Plan

**Status:** Completed / Released

**Date:** 2026-07-14

**Design source:** `docs/superpowers/specs/2026-07-14-v1.3.1-execution-engine-refine-diff-design.md`

## Goal

Add a post-commit Execution Refine workflow that converts changed execution context into a bounded,
validated `PlanDiff`. The user reviews and applies the whole diff atomically without regenerating the
entire plan or changing completed history.

## Non-Negotiable Invariants

1. Existing pre-confirmation `refine` behavior remains unchanged.
2. Execution Refine uses its own durable run and `run_type=execution_refine`.
3. Only the selected project's eligible current execution scope may change.
4. Completed tasks, completed phases, Roadmap, reviews, loops, checkpoints, and history are immutable.
5. Task Assist children are never direct mutation targets.
6. The MVP has no delete operation and no partial Apply.
7. AI output is a proposal; only the backend validator and stored proposal authorize mutations.
8. Task rows and committed TaskTree remain transactionally consistent.
9. DeepSeek is the blocking provider.
10. Feature flags default off until Reviewer and manual acceptance pass.

## Scope Guard

In scope:

- `time_budget`, `progress_recovery`, and `context_change` modes.
- Durable run, snapshot, SSE, cancel, expiry, stale detection, and idempotent Apply.
- `update_task`, `add_task`, `reorder_siblings`, and `set_my_day` operations.
- Project-only entry and in-place diff review.
- 24-case strict Execution Refine Eval.

Out of scope:

- automatic or scheduled refinement;
- cross-project optimization;
- selective operation Apply;
- delete/archive operations;
- whole-plan regeneration;
- personalized memory;
- actual-duration learning;
- AI-controlled phase advancement;
- external calendar or task-system integration.
