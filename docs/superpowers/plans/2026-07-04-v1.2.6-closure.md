# v1.2.6 Portfolio Overview & Answer Layer Closure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the remaining v1.2.6 portfolio information layer and generation-action semantics without adding a backend aggregation API or run-history model.

**Architecture:** Derive portfolio card data from the existing planned tasks and per-thread snapshots through a pure selector. Keep SSE reconnect scoped to the existing `activeRun` by introducing a reconnect nonce, and separate initial `SYNCING` navigation from cancellation so background completion remains intact. The backend planning contract stays unchanged and is verified through existing validator and Eval coverage.

**Tech Stack:** React 18, TypeScript, Zustand, Vitest, React Testing Library, Node test harness, FastAPI, LangGraph, pytest, DeepSeek Eval

**Design spec:** `docs/superpowers/specs/2026-07-04-v1.2.6-closure-design.md`

export interface PortfolioProject {

- Use the current Next Action title.
- When no Next Action:
  - completed current phase: `当前阶段已完成`
  - otherwise: `暂无下一步动作`
- When snapshot is missing, preserve title and return stable fallback labels.

- [ ] **Step 3: Add dismissInitialSync**

Add to `AppStore`:

```ts
dismissInitialSync: () => void;
```

Implementation requirements:

```ts
dismissInitialSync: () => {
  const { activeRun: run, selectedProjectId } = get();
  if (!run || run.runType !== 'initial') return;

  set({
    view: 'board',
    currentViewBucket: 'planned',
    previewMode: null,
    appState: 'INITIAL',
    error: null,
    isRunStalled: false,
  });

  localStorage.setItem('easyplan_view', 'board');
  localStorage.removeItem('easyplan_preview_mode');
  if (selectedProjectId) {
    void get().loadProjectSnapshot(selectedProjectId);
  } else {
    localStorage.removeItem('easyplan_selected_project_id');
  }
  void get().fetchTasks('planned');
},
```

Do not clear:

- `activeRun`
- `threadId`
- `syncRequestId`
- `easyplan_active_run`
- `easyplan_thread_id`

- [ ] **Step 4: Split ActionLayer branches**

Render independent branches instead of:

```ts
appState === 'THINKING' || appState === 'SYNCING'
```

Required behavior:

- `THINKING`: cancellation/dismissal appropriate to the current run.
- `PENDING`: cancel, regenerate, and confirm.
- `ERROR`: retry, return, or start a new intent.
- `SYNCING`: no cancel; new initial run shows `返回全部计划`, refine shows `返回当前计划`.
- initial `THINKING/PENDING` 的本地退出使用 `放弃等待` / `放弃此计划`，不要暗示后端 run 已被取消。
- `isRunStalled`: `继续等待` and `重新连接`; next phase 可保留后端取消，initial 只提供放弃等待。
