# EasyPlan v1.3.0 Task Copilot / Action Coach Implementation Plan

> **Execution rule:** Backend and frontend owners implement in the order below. Do not start UI integration before the proposal, run, and Apply contracts are frozen in OpenAPI. Reviewer performs a cross-layer release review.

**Goal:** Add task-level `start`, `unstick`, and `decompose` assistance with structured DeepSeek proposals, explicit user confirmation, durable run recovery, and transaction-safe local task mutations.

**Architecture:** Store short-lived task-assist runs in a dedicated table, generate a discriminated proposal through DeepSeek, stream progress with an isolated `task_assist` SSE run, and apply only server-authorized mutations. Start updates `start_hint`, unstick updates `fallback_action`, and decompose creates roll-up child tasks without rewriting the plan TaskTree.

**Tech Stack:** FastAPI, Pydantic V2, SQLAlchemy 2.x async, PostgreSQL JSONB, DeepSeek, request-scoped SSE, React 18, TypeScript, Zustand, Vitest, React Testing Library.

**Design source:** `docs/superpowers/specs/2026-07-12-v1.3.0-task-copilot-action-coach-design.md`

**Implementation status (2026-07-13):** Backend and frontend implementation is complete. Local verification records `452 passed` backend tests, `24 passed` Task Assist frontend tests, all legacy `.test.mjs` VM harnesses updated and passed, and successful frontend build/lint. Previously recorded DeepSeek gates are Task Assist `18/18` and Planning `54/54`. All integration gates are green; final Reviewer and manual product acceptance remain before the version is marked Completed.

## Scope Guard

In scope:

- Three modes: `start`, `unstick`, `decompose`.
- Task-level optional user context.
- Durable proposal preview and explicit Apply.
- Independent task-assist SSE lifecycle.
- Roll-up task semantics for decomposed tasks.
- Project and My Day entry points.
- DeepSeek Task Assist Eval with 18 cases.

Out of scope:

- Open-ended chat history.
- Whole-plan refine, rescheduling, Roadmap changes, phase generation, or intent changes.
- Calendar/MCP integration.
- Task templates, “explain this step,” arbitrary 10-minute conversion, and personalization.
- Completed tasks, Group nodes, or practice-loop occurrences.
- Multiple active assist runs for the same task.
- Automatic modification before Apply.

## Product Mutation Matrix

| Mode | Preview | Apply mutation |
| --- | --- | --- |
| `start` | 2–10 minute starter step | update parent `start_hint` metadata |
| `unstick` | 2–3 rescue options | update parent `fallback_action` metadata |
| `decompose` | 2–5 subtask drafts | insert assist children + dependencies, enable roll-up |

No mode may modify another project task, `AgentThread.task_tree`, Roadmap, strategy context, My Day membership, or phase review.

## Manual Product Acceptance

Use tasks from both a project and My Day:

```text
完成商业计划书市场分析章节
联系一位产品经理了解日常工作
整理 N3 词汇摸底结果
发送客户道歉邮件
```

Verify:

1. “帮我开始”给出 2–10 分钟动作，未 Apply 时不改变任务，Apply 后只更新开始提示。
2. “我卡住了”允许补充阻力，给出 2–3 个选项，只保存用户选择的降级动作。
3. “拆得更细”先预览，确认后才创建 2–5 个子任务。
4. 子任务未完成时父任务不能直接完成；全部完成后父任务自动完成。
5. My Day 和项目中的同一任务状态同步。
6. 关闭、取消、刷新、断网、重试和 stale Apply 都有清晰恢复路径；取消失败不会留下空面板或丢失当前 run。
7. Action Coach 不跳转到全局生成页，不改变其他任务或整份计划。
8. Assist child 不能独立加入 My Day，只能随父任务嵌套显示；父任务移出后 child 不再出现在 My Day。
